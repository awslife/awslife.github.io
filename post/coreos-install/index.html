Kubernetes 멀티 노드 설치를 위해 CoreOS 설치 방법을 정리하였다.

CoreOS 설치

CoreOS 홈페이지에서 CoreOS ISO 파일을 다운로드 후 이를 이용하여 CoreOS 설치를 위한 Console로 진입한다. 여기서는 쉬운 설치를 위해서 core 계정의 패스워드를 임시로 변경 후 SSH를 이용한 원격 접속으로 설치를 진행하였다.

암호를 이용한 접속을 위하여 암호 생성

core@localhost ~ $ openssl passwd -1 > cloud-config.yml

core@localhost ~ $ cat cloud-config.yml

Kubernetes Cluster를 위한 환경

#cloud-config

hostname: "k8s-maste1.homelab.local"

coreos:
  etcd2:
    name: "homelab-etcd1"
    # generate a new token for each unique cluster from https://discovery.etcd.io/new?size=3
    # specify the initial size of your cluster with ?size=X
    advertise-client-urls: http://k8s-master1.homelab.local:2379
    initial-advertise-peer-urls: http://k8s-master1.homelab.local:2380
    # listen on both the official ports and the legacy ports
    # legacy ports can be omitted if your application doesn't depend on them
    listen-client-urls: http://0.0.0.0:2379,http://0.0.0.0:4001
    listen-peer-urls: http://k8s-master1.homelab.local:2380,http://k8s-master1.homelab.local:7001
    initial-cluster: homelab-etcd1=http://k8s-master1.homelab.local:2380,homelab-etcd2-02=http://k8s-master2.homelab.local:2380
    initial-cluster-state: new
  units:
    - name: etcd-member.service
      command: start
    - name: systemd-networkd.service
      command: stop
    - name: 00-ens192.network
      runtime: true
      content: |
        [Match]
        Name=ens192

        [Network]
        Address=192.168.11.230/24
        Gateway=192.168.11.1
        DNS=192.168.11.3
    - name: down-interfaces.service
      command: start
      content: |
        [Service]
        Type=oneshot
        ExecStart=/usr/bin/ip link set enp0s25 down
        ExecStart=/usr/bin/ip addr flush dev enp0s25
    - name: systemd-networkd.service
      command: restart
    - name: docker.service
      command: start
    - name: flanneld.service
      drop-ins:
        - name: 50-network-config.conf
          content: |
            [Service]
            ExecStartPre=/usr/bin/etcdctl set /coreos.com/network/config '{ "Network": "10.1.0.0/16" }'

users:
  - name: "core"
    passwd: "blabla"
    groups:
      - "sudo"
      - "docker"
    ssh_authorized_keys:
      - ssh-rsa blablabla

cloud-config가 잘못 설정되었을 경우 조치방법

CoreOS 설치를 위한 cloud-config에 설정이 빠졌을 경우 CoreOS 재 설치를 고려해봐야하는 난감한 상황이 발생한다. 이때 재설치가 아닌 /var/lib/coreos-install/user_data만 수정 후 재 부팅하면 이런 문제를 해결 할 수 있다.

위의 방법은 파일을 직접 수정하는 방법이며, 파일을 직접 수정하게되면 간편하기도 하지만 Validation Check가 동시에 이루어지지 않는다. 더 간단한 방법은 아래의 명령을 이용하는 방법이다.

  ~  coreos-cloudinit -from-file=cloud-config
  ~ 
