CoreOS Cluster의 Kubernetes Master 서버 설치 방법을 정리하였다.

Kubernetes Cluster 구성 정보

  • Master Node Master Node는 3대로 구성하였으며, 각 호스트는 사전에 DNS 구성을 완료하였으며, 모두 2 vCores, 2 GB Memory, 8 GB 스토리지로 구성하였다.
    • k8s-master1.homelab.local : 192.168.11.230
    • k8s-master2.homelab.local : 192.168.11.231
    • k8s-master3.homelab.local : 192.168.11.232
  • Workder Node Worker Node도 마찬가지로 3대로 구성하였으며, 각 호스트는 설치 전에 DNS 구성을 완료하였으며, 모두 4 vCores, 8 GB Memory, 64 GB 스토리지로 구성하였다.
    • k8s-worker1.homelab.local : 192.168.11.233
    • k8s-worker2.homelab.local : 192.168.11.234
    • k8s-worker3.homelab.local : 192.168.11.235

Kubernetes 설치 순서

Kubernetes Cluster 설치 순서는 아래의 프로세스로 진행된다.

  • Kubernetes components 간의 연결을 위한 인증서 생성
  • Kubernetes cluster를 위한 flannel networking 구성
  • Kubernetes Master 노드 설정
  • Kubernetes Worker 노드 설정
  • kubectl 설정
  • 구성 테스트

이번글에서는 Master Node 설정까지 진행해보기로 한다.

Kubernetes components 간의 연결을 위한 인증서 생성

Kubernetes components 간의 암호화된 연결을 위해 만들어야 하는 인증서는 크게 3가지이다.

  • Root CA 공용키 및 개인키

    • ca.pem
    • ca-key.pem

  • API server 공용키 및 개인키

    • k8s-master[1-3].pem
    • k8s-master[1-3]_key.pem

  • Worker node 공용키 및 개인키

    • k8s-worker[1-3].pem
    • k8s-worker[1-3]_key.pem

증서 생성 스크립트를 먼저 작성하였고, 이를 이용하여 인증서를 생성하였다.

awslife@k8sdev:~/Documents/kube-ssl$ cat gen_certs.sh
#!/bin/bash

# Deleting pem, csr, cnf files
rm ./*.pem ./*.csr ./*.cnf ./*.srl

declare -a MASTERS=( "k8s-master1.homelab.local" "k8s-master2.homelab.local" "k8s-master3.homelab.local" )
declare -a MASTERS_IP=( "192.168.11.230" "192.168.11.231" "192.168.11.232" )
declare -a WORKERS=( "k8s-worker1.homelab.local" "k8s-worker2.homelab.local" "k8s-worker3.homelab.local" )
declare -a WORKERS_IP=( "192.168.11.233" "192.168.11.234" "192.168.11.235" )

# Creating TLS certs
echo "Creating CA ROOT**********************************************************************"
openssl genrsa -out ca-key.pem 2048
openssl req -x509 -new -nodes -key ca-key.pem -days 10000 -out ca.pem -subj "/CN=kube-ca"

cat >openssl-master.cnf<<EOF
[req]
req_extensions = v3_req
distinguished_name = req_distinguished_name
[req_distinguished_name]
[ v3_req ]
basicConstraints = CA:FALSE
keyUsage = nonRepudiation, digitalSignature, keyEncipherment
subjectAltName = @alt_names
[alt_names]
DNS.1 = kubernetes
DNS.2 = kubernetes.default
DNS.3 = kubernetes.default.svc
DNS.4 = kubernetes.default.svc.cluster.local
DNS.5 = \$ENV::HOSTNAME
IP.1 = \$ENV::MASTER_IP
EOF

for (( c=1; c<= ${#MASTERS[@]}; c++ ))
do
	echo "creating API Server certs*************************************************************"
  openssl genrsa -out k8s-master${c}_key.pem 2048
	MASTER_IP=${MASTERS_IP[c-1]} HOSTNAME=${MASTERS[c-1]} openssl req -new -key k8s-master${c}_key.pem -out k8s-master$c.csr -subj "/CN=k8s-master$c.homelab.local" -config openssl-master.cnf
	MASTER_IP=${MASTERS_IP[c-1]} HOSTNAME=${MASTERS[c-1]} openssl x509 -req -in k8s-master$c.csr -CA ca.pem -CAkey ca-key.pem -CAcreateserial -out k8s-master$c.pem -days 365 -extensions v3_req -extfile openssl-master.cnf
done

echo "Creating worker nodes"
cat > openssl-worker.cnf<< EOF
[req]
req_extensions = v3_req
distinguished_name = req_distinguished_name
[req_distinguished_name]
[ v3_req ]
basicConstraints = CA:FALSE
keyUsage = nonRepudiation, digitalSignature, keyEncipherment
subjectAltName = @alt_names
[alt_names]
DNS.1 = \$ENV::HOSTNAME
IP.1 = \$ENV::WORKER_IP
EOF

for (( c=1; c<= ${#WORKERS[@]}; c++ ))
do
   openssl genrsa -out k8s-worker${c}_key.pem 2048
   WORKER_IP=${WORKERS_IP[c-1]} HOSTNAME=${WORKERS[c-1]} openssl req -new -key k8s-worker${c}_key.pem -out k8s-worker$c.csr -subj "/CN=k8s-worker$c.homelab.local" -config openssl-worker.cnf
   WORKER_IP=${WORKERS_IP[c-1]} HOSTNAME=${WORKERS[c-1]} openssl x509 -req -in k8s-worker$c.csr -CA ca.pem -CAkey ca-key.pem -CAcreateserial -out k8s-worker$c.pem -days 365 -extensions v3_req -extfile openssl-worker.cnf
done

echo "Creating Admin certs********************************************************"
openssl genrsa -out admin-key.pem 2048
openssl req -new -key admin-key.pem -out admin.csr -subj "/CN=kube-admin"
openssl x509 -req -in admin.csr -CA ca.pem -CAkey ca-key.pem -CAcreateserial -out admin.pem -days 365

CoreOS 환경 설정

CoreOS 버전이 올라감에 따라 친숙한 cloud-config에서 Ignition으로 변경되었다. 다행히 cloud-config에서 사용되던 yaml 파일을 coreos-ct를 통하여 Ignition으로 일괄 변경 가능하다. Mac에서는 brew를 통하여 설치 가능하다.

Master 설치 진행

Master Node 1번에 대해 설치를 진행해 보았다. k8s-master1에 대한 설정은 k8s-master1와 동일하며 2,3번의 경우 IP, Hostname, PEM만 다르고 나머지는 동일하다. 만들어진 Yaml 파일을 ct 명령을 사용하여 Ignition Format으로 변경하자. (ct 명령에 대한 사용법 및 설치 방법은 아래 Reference에 자세하게 정리되어 있다.)
